#### 2.4.2.1.生成报送数据

1、  每月1号凌晨4点执行定时任务，按照《应收账款债务人业务信息》所需字段，去业务表拉起数据，并按照债务人加授信主体为维度对数据进行加工。

（1）取数的字段及处理逻辑如下：

[应收账款债务人业务信息报送](https://doc.weixin.qq.com/sheet/e3_AR4AHQY_AHQCNrjLeFoJAQS6LHoxy?scode=ABAA4gf_AAw9QFhCRzAR4AHQY_AHQ\&tab=BB08J2)

（2）在查询业务数据，并进行加工过程中，需要过滤到以下数据（单独建一张表，打上标签，后续可灵活增加）：

● 标签一：测试企业（根据企业id进行过滤）

● 标签二：指定资产（根据资产编号进行过滤）

（3）在生成报送数据时，默认只对有过融资（融资中、融资成功、融资失败）的原始凭证进行报送。

（4）若同一债务人对应多个授信主体（如企业既有自有额度又有集团额度），需要分开报送。

（5）债务人所属集团处理：

先判断债务人是否有在组织架构树中。

● 若债务人存在于组织架构树，取组织架构树一级单位为应收账款债务人所属集团，并调dabss接口获取集团统一社会信用代码（若系统中有集团统一社会信用代码则无需调dbass获取）。

● 若债务人不存在于组织架构树中，则根据应收账款债务人统一社会信用代码调dbass接口获取应收账款债务人所属集团及应收账款债务人所属集团社会信用代码，并将查询结果保存到数据表中，下次直接取表中数据。

● 若接口获取不到应收账款债务人所属集团信息，则应收账款债务人所属集团、应收账款债务人所属集团社会信用代码赋值为“无”。

（2）【数据报送审核岗】可对报送删除申请进行审核。若选择审核通过，系统会调用互金协会【报送删除】接口进行报送删除，数据状态变为”报送删除“。若选择审核不通过，数据状态为”已报送“。

● 数据报送&审核：

（1）【数据报送经办岗】点击“数据报送”，系统会弹出确认提示，确认之后，提交数据报送申请，此时报送状态变为“数据报送审核中”。

（2）【数据报送审核岗】可对数据报送进行审核。若选择审核通过，系统调用协会报送入库接口进行数据报送，报送状态会变为“报送中”。若选择审核不通过，报送状态更新为审核前的状态。

（3）可针对报送状态为“待报送”、“报送失败”、“报送删除”数据进行数据报送。

● 查看日志：

（1）点击“查看日志”，系统会弹出页面，展示该条报送的操作日志（包括操作内容、操作人、操作时间三个字段），按照操作时间从近到远进行展示。

| 操作内容     | 展示说明                                                      |

| 重新生成报送数据 | 指手动点击重新生成按钮，生成报送数据。此时操作人显示点击重新生成按钮的用户姓名，操作时间展示点击重新生成按钮的时间 |
| 数据调整申请   | 指用户提交数据调整申请。此时操作人显示点击数据调整提交按钮的用户姓名，操作时间展示提交时间             |
| 数据调整审核   | 指用户进行数据调整审核。此时操作人显示审核时的用户姓名，操作时间展示审核提交时间                  |
| 数据报送申请   | 指用户提交数据报送申请。此时操作人显示点击数据报送提交按钮的用户姓名，操作时间展示提交时间             |
| 数据报送审核   | 指用户进行数据报送审核。此时操作人显示审核时的用户姓名，操作时间展示审核提交时间                  |
